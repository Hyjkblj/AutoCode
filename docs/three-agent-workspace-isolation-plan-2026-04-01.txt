四 Agent 任务划分（工作区严格独立，移动端独立开发）
日期：2026-04-01
最后更新：2026-04-05

目标
- 将当前主线剩余问题拆分为 4 个互不干扰的开发任务（其中移动端单独一条线）。
- 每个 Agent 在独立 worktree + 独立分支开发，禁止跨路径改动，避免工作区污染。

一、统一隔离规则（所有 Agent 必须遵守）
1. 每个 Agent 使用独立 worktree：
   - git worktree add ..\AutoCode-a1 -b fix/a1-control-plane-authz master
   - git worktree add ..\AutoCode-a2 -b fix/a2-shared-protocol-contract master
   - git worktree add ..\AutoCode-a3 -b fix/a3-java-agent-artifact-payload master
   - git worktree add ..\AutoCode-a4 -b feat/a4-mobile-visual-app master
2. 禁止在主工作区直接开发。
3. 禁止使用 git add .，只能按文件精确 add。
4. 提交前必须执行 git status --short，若出现白名单外文件，必须先还原再提交。
5. 禁止提交临时文件（__pycache__、*.pyc、日志、IDE 文件）。
6. 交叉路径禁改（移动端必须区别于 Agent-1/2/3）：
   - Agent-1/2/3：禁止修改移动端白名单路径（mobile-app/**）。
   - Agent-4：禁止修改 control-plane-spring/**、shared-protocol/**、pc-agent-java/**、python-agent/**。

二、Agent 角色定义
- Agent-1：Control Plane Security Agent（控制面安全 + 策略引擎 + TLS）
  白名单：control-plane-spring/src/main/** + control-plane-spring/src/test/**
- Agent-2：Shared Protocol Contract Agent（协议契约 + Java 基础设施 + 沙箱 HTTP 接口）
  白名单：shared-protocol/src/main/** + shared-protocol/src/test/** + pc-agent-java/sandbox-server/**
- Agent-3：Java/Python Agent Runtime Agent（Java Agent 运行时重构 + Python Agent 全部）
  白名单：pc-agent-java/src/main/** + pc-agent-java/src/test/** + python-agent/**
- Agent-4：Mobile Visual App Agent（移动端全部）
  白名单：mobile-app/**

三、主控最终门禁
1. git status --short 必须干净。
2. Java 回归：
   - mvn -B -q -pl shared-protocol,control-plane-spring,pc-agent-java -am test
3. Python Agent 回归（PY-AG-01 合入后）：
   - cd python-agent && pytest -q
4. 移动端回归：
   - cd mobile-app && .\gradlew.bat :app:assembleDebug
