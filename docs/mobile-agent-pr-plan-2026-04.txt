负责人：Agent-4
优先级：P0（AI Agent 接入后最核心的展示层）
分支建议：feat/a4-mob-websocket-events
依赖：无（控制平面 WebSocket 已就绪）

目标
- 替换当前 HTTP 轮询（每 2.5s），改为 WebSocket 订阅实时事件流。
- 用户在任务详情页能实时看到 AI Agent 的每一步工作过程。
- 断线后自动重连，并通过 lastSeq 补回断线期间的事件。

技术方案
- 使用 OkHttp WebSocket（已在 build.gradle.kts 中引入）。
- 连接地址：ws://{baseUrl}/ws（STOMP over WebSocket）。
- 订阅 topic：/topic/tasks/{taskId}。
- 首次连接时携带 lastSeq=0，断线重连时携带最后收到的 seq。
- 补流接口：GET /api/v1/tasks/{taskId}/events?lastSeq={n}（HTTP 拉取历史）。

新增/变更内容
- network/WebSocketClient.kt（新增）
  - connect(taskId, lastSeq, onEvent, onDisconnect)
  - disconnect()
  - 自动重连（指数退避，最大 30s）
- network/EventDto.kt（新增）
  - eventId, taskId, type, payload, seq, timestamp
- AppViewModel.kt（变更）
  - 新增 subscribeTaskEvents(taskId)
  - 新增 unsubscribeTaskEvents()
  - 移除 startRemotePolling() 中的状态轮询（保留任务列表轮询）
- AppUi.kt（变更）
  - TaskDetailTab：进入页面时订阅，离开时取消订阅
  - 事件流展示区：按 seq 顺序渲染每条事件

事件类型展示规则
- TASK_STARTED       → "任务已开始"
- ASSISTANT_OUTPUT   → 显示 payload.message（AI Agent 工作日志）
- TOOL_START         → "执行工具：{tool}  命令：{command}"
- TOOL_END           → "工具完成：{status}  耗时：{execMs}ms"
- FILE_PATCH_PREVIEW → "代码变更预览（点击展开）"
- APPROVAL_REQUIRED  → 触发审批卡片弹出（MOB-02 实现）
- APPROVAL_RESULT    → "审批结果：{decision}"
- TASK_DONE          → "任务完成"
- TASK_FAILED        → "任务失败：{reason}"

验收
- 连接控制平面，创建任务，任务详情页实时出现事件流
- 断开网络后重连，事件流从断点续流（lastSeq 正确）
- 离线模式（无 baseUrl）降级为本地模拟 ticker，不报错

目标
- 收到 APPROVAL_REQUIRED 事件时，弹出审批卡片。
- 卡片展示完整审批上下文（command / cwd / action / riskScore / reason）。
- 用户点击批准/拒绝后，调用控制平面审批 API。
- 审批结果通过 WebSocket 回传，卡片自动关闭。

技术方案
- 审批卡片使用 AlertDialog 或 BottomSheet（Material3）。
- 审批 API：POST /api/v1/tasks/{taskId}/approval
  - body: { approvalId, decision: "approve"|"reject", comment }
- 审批超时（approvalTimeoutSeconds）倒计时展示，超时自动关闭卡片。

新增/变更内容

  - TaskDetailTab：监听 pendingApproval 状态，弹出 ApprovalCard
- AppViewModel.kt（变更）
  - 新增 pendingApproval: StateFlow<ApprovalRequest?>
  - 新增 submitApproval(taskId, approvalId, decision, comment)
  - 收到 APPROVAL_REQUIRED 事件时更新 pendingApproval
  - 收到 APPROVAL_RESULT 事件时清空 pendingApproval
- network/ControlPlaneClient.kt（变更）
  - 新增 submitApproval(baseUrl, token, taskId, approvalId, decision, comment)
- Models.kt（变更）
  - 新增 ApprovalRequest(approvalId, taskId, action, command, cwd, riskScore, reason, timeoutSeconds)

- 卡片展示 command/cwd/riskScore 等上下文信息

- 倒计时结束后卡片自动关闭（超时处理）
