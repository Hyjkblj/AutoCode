四 Agent 任务划分（工作区严格独立，移动端独立开发）
日期：2026-04-01

目标
- 将当前主线剩余问题拆分为 4 个互不干扰的开发任务（其中移动端单独一条线）。
- 每个 Agent 在独立 worktree + 独立分支开发，禁止跨路径改动，避免工作区污染。

一、统一隔离规则（所有 Agent 必须遵守）
1. 每个 Agent 使用独立 worktree：
   - git worktree add ..\AutoCode-a1 -b fix/a1-control-plane-authz master
   - git worktree add ..\AutoCode-a2 -b fix/a2-shared-protocol-contract master
   - git worktree add ..\AutoCode-a3 -b fix/a3-java-agent-artifact-payload master
   - git worktree add ..\AutoCode-a4 -b feat/a4-mobile-visual-app master
2. 禁止在主工作区直接开发。
3. 禁止使用 git add .，只能按文件精确 add。
4. 提交前必须执行 git status --short，若出现白名单外文件，必须先还原再提交。
5. 禁止提交临时文件（__pycache__、*.pyc、日志、IDE 文件）。
6. 交叉路径禁改（移动端必须区别于 Agent-1/2/3）：
   - Agent-1/2/3：禁止修改移动端白名单路径（mobile-app/**、app-mobile/**、apps/mobile/**）。
   - Agent-4：禁止修改 control-plane-spring/**、shared-protocol/**、pc-agent-java/**（如需能力支持，必须拆分为对应 Agent 的 PR 先合入）。

二、Agent-1（控制面权限修复，P0）
角色
- Control Plane Security Agent

任务
1. 修复项目级访问控制语义，确保非成员访问 artifacts 返回 404。
2. 保持管理员/运维角色策略可解释、可配置，不放大越权面。
3. 修复并通过对应集成测试（ArtifactsIntegrationTest 相关失败用例）。

允许修改路径（白名单）
- control-plane-spring/src/main/**
- control-plane-spring/src/test/**

禁止修改路径
- shared-protocol/**
- pc-agent-java/**
- docs/**
- scripts/**
- pom.xml（仓库根）

验收命令
- mvn -pl shared-protocol,control-plane-spring -am -DskipTests compile
- mvn -pl shared-protocol,control-plane-spring -am -Dtest=ArtifactsIntegrationTest test

三、Agent-2（协议契约收口，P1）
角色
- Shared Protocol Contract Agent

任务
1. 收口 ARTIFACT_READY 契约（schema/DTO/example/validator 一致性）。
2. 确保 eventVersion=1 下新增事件的兼容策略清晰且可测试。
3. 清理示例文件与测试资源的不一致（路径与内容一致）。

允许修改路径（白名单）
- shared-protocol/src/main/**
- shared-protocol/src/test/**
- shared-protocol/src/main/resources/**
- shared-protocol/src/test/resources/**
- shared-protocol/COMPATIBILITY.md
- shared-protocol/pom.xml

禁止修改路径
- control-plane-spring/**
- pc-agent-java/**
- docs/**
- scripts/**
- pom.xml（仓库根）

验收命令
- mvn -pl shared-protocol test

四、Agent-3（Java Agent 契约对齐，P1）
角色
- Java Agent Runtime Agent

任务
1. 将 Java Agent 的 ARTIFACT_READY payload 生成逻辑与 shared-protocol 对齐。
2. 确保上报事件字段（eventId/eventVersion/payload）与控制面校验兼容。
3. 增补或修复 pc-agent-java 对应单测，避免回归。

允许修改路径（白名单）
- pc-agent-java/src/main/**
- pc-agent-java/src/test/**
- pc-agent-java/pom.xml
- pom.xml（仅 dependencyManagement 必要改动）

禁止修改路径
- control-plane-spring/**
- shared-protocol/**
- docs/**
- scripts/**

验收命令
- mvn -pl shared-protocol,pc-agent-java -am test

五、Agent-4（移动端可视化 App，P1）
角色
- Mobile Visual App Agent

任务
1. 构建移动端可视化 App：登录、项目选择、自然语言输入、任务进度、产物预览/发布入口、历史与版本。
2. 与控制面/协议契约对齐，但禁止直接修改 control-plane-spring/shared-protocol/pc-agent-java（通过提需求给对应 Agent 来补齐缺口）。
3. 提供最小可用的端到端手动测试路径（至少能完成一次“发起生成→看到进度→看到产物入口”）。

允许修改路径（白名单）
- mobile-app/**
- app-mobile/**
- apps/mobile/**

禁止修改路径
- control-plane-spring/**
- shared-protocol/**
- pc-agent-java/**
- docs/**
- scripts/**
- pom.xml（仓库根）

验收命令
- 以移动端模块自身的构建/测试命令为准（在 mobile-app/ 内执行），并要求 git status --short 干净

六、合并顺序（主控执行）
1. 先合 Agent-2（协议）到 master。
2. 再合 Agent-1（控制面权限修复）。
3. 再合 Agent-3（Java Agent 契约对齐）。
4. 最后合 Agent-4（移动端可视化 App）。

七、主控最终门禁
1. git status --short 必须干净。
2. Java 回归：
   - mvn -B -q -pl shared-protocol,control-plane-spring,pc-agent-java -am test
3. 如恢复 Python Runner 后，再补：
   - cd claudecode-runner && python -m pytest -q
4. 移动端回归：
   - 在移动端模块内执行其构建/测试命令（至少能完成一次端到端手动验证流程）

八、提交模板（每个 Agent）
1. git status --short
2. git add <白名单内文件>
3. git commit -m "fix(<scope>): <summary>"
4. git status --short（确认仅剩本分支预期变更或已干净）
